Use frames apenas quando não houver outras alternativas

Um uso para frames é esconder a URI atual. Mas existem aplicações mais interessantes:

  • Wizards
  • Pop-Ups
  • Troca de dados entre subcontextos

Para obter sucesso no uso de frames, algumas regras devem ser seguidas:

  • Cada documento é o mais independente possível. Eles devem trocar mensagens entre si através de funções em escopo público do frame em questão.
  • Um documento jamais se refere diretamene aos id's do outro.
  • Todos os frames devem ter nomes.
  • Em aplicações completas, a navegação entre os frames deve ser feita através do top.

Frames, em pleno 2010, são desrecomendados principalmente por causa da ineficiência das ferramentas de debug existentes para javascript/html.
